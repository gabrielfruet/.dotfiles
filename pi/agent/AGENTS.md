# Agent Guidelines

## Actions
- User asks to act (edit, write, lint, fix, run, create, delete) → just do it
- User asks to analyze (see, check, look, find, read) → just analyze, don't act

## Code
- KISS - keep it simple stupid
- Reliable over clever
- Docstrings, not block comments
- Explicit over implicit
- Small functions
- Minimal deps
- Composition over inheritance
- Test code
- Handle errors
- Explicit types
- Simple solutions first
